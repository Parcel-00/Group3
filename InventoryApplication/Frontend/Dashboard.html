Parcel Dashboard

Standalone interface prototype. Buttons respond, but logging and scanner integration are intentionally not yet enabled.

Status

Front-end only. No live camera feed, no barcode decoding, and no manifest data source at this stage.

About

This is a prototype interface for a shipping container manifest scanner and logger. In the full system, camera-based detection would identify the container (LEGO/wooden-block proxy), barcode reading would capture IDs, and damage/anomaly detection would record defects and compute an anomaly percentage in 20% increments.

Integration points (future): camera input, barcode decoding, manifest store, and analytics.

View Logger

This view will eventually display the logged manifests after containers are detected and scanned. For now, it is a placeholder to verify navigation and layout.

No manifest data available.
When integrated, this panel will show container IDs, header fields, inventory lines, and recorded anomaly scans.

Login

This is a non-functional login stub. It responds to input and button presses, but does not authenticate against any backend service at this stage.

Admin Login

Admin access requires a special code known only to group members. This prototype checks the code locally and, if correct, navigates to an admin placeholder screen. No backend access is available yet.

Admin Panel (placeholder)

This screen represents where administrative functions would appear once integrated (for example: configuration, audit exports, user management, and direct access to inspection results).

Admin features are not integrated.
Expected behaviour (future): authenticated access to backend-only functions and restricted operational controls.
